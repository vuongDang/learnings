# AI control

How to prevent untrusted model to cause harm?

In the context of having AIs being smarter than humans 

## Ideas 

- Trusted model checking on Untrusted model 
  - dumber judge model but trusted check on untrusted smart model 
  - trusted monitoring: judges flags suspicious outputs
  - trusted editing: judge sanitizes the target outputs
- Untrusted model monitoring each other
  - Have to be wary of hidden collusion between untrusted
- Use humans for consequential decision
  - human fatigue may let through harmful requests
  - in the race for performance human is the bottleneck
- Separate task into smaller chunks 
  - different instances of untrusted accomplish them 
  - trusted reassemble them
  - good to prevent complex multi-step sabotage
- Input/Output filtering
  - using classifiers to block harmful input/output
  - constitutional classifier are emerging
    - trained over a large dataset of synthetic data
    - data was generated by a model following a constitution
  - susceptible to jailbreaks
  - classifiers can be biased depending on the training data
  - it's hard to create good rules to follow

## Limitations

- We have to hope that AI labs are willing to use control measures
- Can we expect super intelligent AIs to be controlled?
